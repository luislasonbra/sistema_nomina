# 6a. Realizacion de Caso de Uso - Procesar Nomina

## Caso de uso seleccionado

Procesar nomina autenticada

## Objetivo

Permitir que un usuario autenticado seleccione un periodo de nomina, calcule los montos de pago de los empleados activos, revise el detalle y procese la nomina de forma definitiva.

## Actores

- Usuario autenticado
- Sistema de Nomina RD

## Precondiciones

- El usuario ha iniciado sesion.
- Existen empleados activos.
- Existe al menos un periodo creado.
- El periodo seleccionado no esta cerrado.

## Postcondiciones

- Se crea una nomina procesada para el periodo.
- Se generan los detalles por empleado.
- El periodo cambia a estado `procesado`.
- La informacion queda disponible en historial y recibos.

## Flujo principal

1. El usuario inicia sesion.
2. El usuario entra al modulo `Procesar Nomina`.
3. El sistema carga periodos, departamentos y empleados.
4. El usuario selecciona un periodo.
5. El usuario opcionalmente selecciona un departamento.
6. El usuario pulsa `Calcular`.
7. El sistema valida el periodo.
8. El sistema consulta empleados activos.
9. El sistema calcula salario proporcional, deducciones obligatorias y descuentos individuales.
10. El sistema muestra el detalle del calculo y los totales.
11. El usuario revisa la informacion.
12. El usuario pulsa `Procesar`.
13. El sistema elimina una nomina previa del mismo periodo si existe.
14. El sistema crea el encabezado de nomina.
15. El sistema crea los detalles por empleado.
16. El sistema actualiza el estado del periodo a `procesado`.
17. El sistema confirma el procesamiento.

## Flujos alternos

### A1. Usuario no autenticado

1. El usuario intenta acceder a la API.
2. El sistema responde con error `401`.
3. La interfaz muestra la pantalla de login.

### A2. No se selecciona periodo

1. El usuario pulsa `Calcular`.
2. El sistema muestra un mensaje indicando que debe seleccionar un periodo.

### A3. Periodo cerrado

1. El usuario intenta procesar un periodo cerrado.
2. El sistema bloquea la operacion.
3. El sistema muestra el mensaje correspondiente.

### A4. No hay empleados activos

1. El sistema no encuentra empleados activos para el filtro seleccionado.
2. El sistema muestra una tabla vacia con totales en cero.

## Responsabilidades por capa

### Interfaz

- Solicitar login.
- Mostrar tabs `Procesar`, `Historial` y `Recibo`.
- Capturar periodo y departamento.
- Presentar resultados y mensajes al usuario.

### Backend

- Validar sesion.
- Validar datos de entrada.
- Calcular deducciones y netos.
- Persistir nomina y detalles.

### Persistencia

- Leer usuarios, periodos, empleados y descuentos.
- Guardar nomina procesada.
- Guardar detalle de nomina.
